# 基于微服务的智能网络优化系统

## 一、项目背景

本项目实现了一个端到端的智能网络优化系统，采用微服务架构，支持业务感知、异常检测与智能编排。

## 三、微服务设计

### 3.1 服务清单

| 服务名 | 容器名 | 端口 | 功能描述 | 入口文件 |
|--------|--------|------|----------|----------|
| 用户中心 | data-driven-usercenter | 8003 | 用户注册/登录认证 | server.py |
| API网关 | data-driven-gateway | 8001 | 请求路由、队列写入、结果轮询 | services/gateway.py |
| 业务感知 | data-driven-perception | - | 消费队列、异常检测、写入数据库 | services/perception_service.py |
| 智能编排 | data-driven-orchestration | - | PPO强化学习决策、结果输出 | services/orchestration_service.py |

### 3.2 基础设施

| 组件 | 容器名 | 端口 | 用途 |
|------|--------|------|------|
| MySQL | data-driven-mysql | 3306 | 数据持久化 |
| Redis | data-driven-redis | 6379 | 消息队列 |

## 七、部署步骤

### 7.1 服务器部署

```bash
# 1. 进入 docker 目录
cd /var/data-driven_2/docker

# 2. 构建基础镜像
docker build -f Dockerfile.base -t data-driven-base:latest ..

# 3. 启动所有服务
docker-compose up -d

# 4. 查看状态
docker-compose ps
```

### 7.2 本地开发启动

```bash
# 1. 启动 MySQL 和 Redis
docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=QosRoot@123 --name mysql mysql:8
docker run -d -p 6379:6379 --name redis redis:7-alpine

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动各服务（各开一个终端）
python server.py              # 用户中心 8003
python services/gateway.py    # API网关 8001
python services/perception_service.py
python services/orchestration_service.py
```

## 九、数据流

```
1. 客户端 → Gateway → 写入 Redis 队列(perception_queue)
2. Perception → 消费队列 → 异常检测 → 写MySQL → 写入编排队列
3. Orchestration → 消费编排队列 → PPO决策 → 写结果到MySQL
4. 客户端 → Gateway → 轮询结果返回
```
