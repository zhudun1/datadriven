## 数据流详解

### 1. 用户登录流程

```
┌────────┐    POST /login     ┌─────────────┐    SQL     ┌────────┐
│  前端  │ ────────────────▶ │ server.py   │ ─────────▶ │ MySQL  │
│        │    {email,        │ (端口 8003) │  SELECT    │        │
│        │     password}     │             │            │        │
└────────┘                    └─────────────┘            └────────┘
      │                         │                          │
      │◀───────────────────────│                          │
      │     {result,           │                          │
      │      token, role}       │                          │
      ▼                         ▼                          ▼
  保存 token                验证成功                  查询用户
  到 localStorage           返回结果                   验证密码
```

**请求路径**：`前端 → server.py:8003 → MySQL`

**关键代码**：
```javascript
// frontend/js/api.js
await request("/login", {...})

// server.py (do_POST)
if self.path == "/login":
    x.execute("SELECT user_id, role FROM t_user WHERE username=%s AND password_hash=%s", ...)
```

### 2. 编排请求流程 (完整流水线)

```
┌────────┐   POST /pipeline   ┌─────────────┐
│  前端  │ ─────────────────▶ │   Gateway   │
│        │   FormData:        │ (端口 8001) │
│        │   businessImage    │             │
│        │   pointCloudPath   └──────┬──────┘
│        │                          │
└────────┘                   ┌──────┴──────┐
                             │             │
                             ▼             ▼
                    写入 business_awareness  发布到 Redis
                    t_industrial_data       rq_perception
                             │             (消息队列)
                             │             (task_id)
                             ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                           消息队列 (Redis)                               │
│                                                                         │
│   [感知队列] ──────────────────────────▶ [编排队列] ──────────────▶ ... │
│   rq_perception                              rq_orchestration           │
│   task_id=123                                 task_id=123              │
│   {rgb_path, pcd_path}                      {qos_vector, ...}          │
└────────────────────────────────────────────┬────────────────────────────┘
                                             │
          ┌──────────────────────────────────┼──────────────────────────┐
          │                                  │                          │
          ▼                                  ▼                          ▼
┌─────────────────────┐          ┌─────────────────────┐     ┌─────────────────────┐
│   感知服务          │          │   编排服务          │     │   基础设施服务      │
│ (perception_service)│          │(orchestration_service)│     │   (待实现)          │
│                     │          │                     │     │                     │
│ 1. 加载模型         │          │ 1. PPO 决策         │     │ 部署 VNF            │
│ 2. 异常检测         │          │ 2. 环境执行         │     │ 下发流表            │
│ 3. QoS 翻译         │          │ 3. 写编排日志       │     │                     │
│                     │          │                     │     │                     │
│ 输入: RGB + 点云    │          │ 输入: QoS 向量      │     │ 输入: 编排决策      │
│ 输出: qos_vector    │          │ 输出: 编排结果      │     │ 输出: 执行结果      │
└─────────┬───────────┘          └─────────┬───────────┘     └─────────────────────┘
          │                              │
          │  发布到编排队列               │  推送结果到结果队列
          ▼                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                          结果队列 (Redis)                               │
│                          rq_results                                     │
│                          {task_id, result: {...}}                      │
└────────────────────────────────────────────┬────────────────────────────┘
                                           │
                                           ▼
                    ┌────────────────────────────────────────────────┐
                    │              Gateway 轮询等待结果                │
                    │         mq.get_result(task_id, timeout=120)     │
                    └────────────────────────────────────────────────┘
                                           │
                                           ▼
                    ┌────────────────────────────────────────────────┐
                    │              返回结果给前端                     │
                    │    {status, vnf_node, link_path, qos_ok, ...} │
                    └────────────────────────────────────────────────┘
```

**详细步骤**：

| 步骤 | 服务 | 操作 | 数据库/队列 |
|------|------|------|-------------|
| 1 | 前端 | 提交 FormData | - |
| 2 | Gateway | 写 `t_industrial_data` | MySQL |
| 3 | Gateway | 发布到感知队列 | Redis `rq_perception` |
| 4 | 感知服务 | 消费消息，加载模型推理 | - |
| 5 | 感知服务 | 异常检测 → QoS翻译 | - |
| 6 | 感知服务 | 写 `t_perception_log` | MySQL |
| 7 | 感知服务 | 发布到编排队列 | Redis `rq_orchestration` |
| 8 | 编排服务 | 消费消息，PPO决策 | - |
| 9 | 编排服务 | 写 `t_orchestration_log` | MySQL |
| 10 | 编排服务 | 推送结果到结果队列 | Redis `rq_results` |
| 11 | Gateway | 轮询获取结果 | - |
| 12 | Gateway | 返回给前端 | - |

## 数据格式示例

```javascript
{
  businessImage: File,      // RGB 图像文件
  pointCloudPath: "path/to/pcd.tiff",  // 点云路径
  resourceRequest: {        // 资源需求 (可选)
    resourceId: "node-0",
    vcpu: 8,
    memory: 16,
    storage: 100,
    bandwidth: 1000
  }
}
```

### 2. QoS 向量 (感知服务生成)

```python
# 4 维向量: [带宽, 延迟, 优先级, 丢包率]

```

## 服务端口汇总

| 服务 | 端口 | 用途 |
|------|------|------|
| 前端静态服务器 | 8003 | 静态页面 + 登录/注册/资源查询 |
| API Gateway | 8001 | 编排请求入口 |
| 感知服务 | 后台 | 异常检测 + QoS翻译 |
| 编排服务 | 后台 | PPO决策 |
| MySQL | 3306 | 数据持久化 |
| Redis | 6379 | 消息队列 |
