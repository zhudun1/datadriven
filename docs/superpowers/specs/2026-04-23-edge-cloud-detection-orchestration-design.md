# 边缘云协同智能检测编排系统设计

**日期**: 2026-04-23

## 一、项目背景

### 1.1 业务场景

**场景**: 工厂生产线上产品缺陷检测（工业质检）

**核心问题**: 传统检测系统无论是否有异常都维持高资源消耗，成本高昂

**解决思路**: 守护模式（低频低耗）与 告警模式（高频高耗）按需切换

### 1.2 业务流程

```
守护模式（低频） → 检测到异常 → 告警模式（高频） → 编排响应 → 恢复守护
```

| 模式 | 检测频率 | 传输带宽 | 计算资源 | 响应动作 |
|------|---------|---------|--------|---------|
| 守护模式 | 1次/10分钟 | 低（缩略图<100KB） | 1个模型 | 本地存储 |
| 告警模式 | 30次/秒 | 高（原始视频>10MB/s） | 4个模型并行 | 声光告警+存证+停机 |

### 1.3 告警模式触发响应动作

1. **声光告警**: 本地响铃/闪灯报警
2. **高频检测**: 多模型并行部署（4个模型同时推理）
3. **多链路传输**: 原始视频多路冗余传输（保障可靠性）
4. **自动保护**: 触发自动停机/遮挡保护
5. **证据存储**: 证据视频本地+云端双存储

## 三、核心设计

### 3.1 守护模式 vs 告警模式

**守护模式 (GUARD)**:
- 检测频率: 1次/10分钟
- 上传数据: 缩略图 (<100KB)
- ���型数量: 1个（轻量模型）
- 网络要求: 低带宽，低优先级
- 成本消耗: 最小化

**告警模式 (ALARM)**:
- 检测频率: 30次/秒
- 上传数据: 原始视频 (>10MB/s)
- 模型数量: 4个（多模型并行）
- 网络要求: 高带宽，低延迟
- 资源消耗: 按需弹性扩展

### 3.2 状态机设计

```
┌─────────┐  异常分数>阈值   ┌──────────┐  人工确认/超时   ┌─────────┐
│ GUARD   │ ──────────────▶  │ ALARM   │ ──────────────▶  │ GUARD   │
│ (守护)  │                 │ (告警)  │                 │ (守护)  │
└─────────┘                 └──────────┘                 └─────────┘

状态转换条件:
- GUARD → ALARM: anomaly_score >= HIGH_THRESHOLD (0.7)
- ALARM → GUARD: 人工确认恢复 或 持续N分钟无异常
```

### 3.3 QoS需求映射

| 状态 | 带宽 | 时延 | 优先级 | 丢包率 | CPU需求 |
|------|-----|-----|-----|-------|-------|
| GUARD | 10Mbps | 200ms | 0 | 20% | 0.2核 |
| ALARM | 80Mbps | 25ms | 6 | 1% | 0.8核×4 |

## 四、核心价值

### 4.1 智能决策 (选项C)

**解决的问题**: 在边缘云环境中，如何自动判断最佳部署位置和传输策略？

**实现方式**:
1. 自动判断当前应该处于守护模式还是告警模式
2. 自动决定在哪个边缘节点部署检测模型
3. 自动选择最优传输链路
4. 最大化资源利用率 + 最小化成本

### 4.2 价值对比

| 指标 | 传统方案 | 本方案 |
|------|---------|-------|
| 带宽消耗 | 恒定 10MB/s | 平均 0.1MB/s（守护） |
| 计算成本 | 恒定 4核 | 平均 0.5核（守护） |
| 异常召回率 | 95% | 99%（高频检测） |
| 响应延迟 | <10秒 | <1秒 |

## 六、接口设计

### 6.1 状态切换请求

```python
class OrchRequest(BaseModel):
    mode: str = "GUARD"  # "GUARD" 或 "ALARM"
    anomaly_score: Optional[float] = None
    duration: Optional[int] = None  # 告警持续时间(秒)
```

### 6.2 编排响应

```python
class OrchResponse(BaseModel):
    mode: str  # 当前模式
    target_nodes: List[int]  # 部署节点
    video_links: List[int]  # 传输链路
    actions: List[str]  # 触发动作: ["alarm_on", "video_record", "machine_stop"]
    expected_cost: float  # 预期成本
```
